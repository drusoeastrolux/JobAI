03 — Backend
Express Server
src/server.ts
The core of the app. Reads the incoming request, prepends the system prompt (Ruixen's persona + resume + job context), and calls OpenRouter with the full assembled message array. The API key lives here and never reaches the browser.
system prompt inject resume append job context append API key guard
Core logic
HTTPS to OpenRouter
openrouter.ai/api/v1 · Authorization header · stream: true
04 — Gateway
OpenRouter
openrouter.ai/api/v1/chat/completions
Validates the API key, reads the model ID from the request, and routes the conversation to the correct AI provider. Acts as a single unified gateway so you don't need separate accounts for each model vendor.
key validation model routing provider dispatch
Gateway
Dispatched to model
openai/gpt-oss-120b:free · set in .env
05 — Model
GPT-OSS 120B
openai/gpt-oss-120b:free via OpenRouter
Reads the full assembled context — system prompt, resume, job posting, and conversation history — and generates a response token by token. Each token streams back through OpenRouter and Express to the browser in real time.
reads full context token-by-token output SSE chunks
Inference
Streamed back
Model → OpenRouter → Express → Browser · data: {content}\n\n
Response rendered
Each token appends to the message bubble live. Once complete, the full reply is saved to localStorage so it persists across page refreshes.
